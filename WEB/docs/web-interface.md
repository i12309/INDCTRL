# Web-интерфейс INDCTRL

INDCTRL использует единый Django-интерфейс. После входа пользователь попадает в
общую оболочку с sidebar-меню. Разделы скрываются по правам, а доступ дополнительно
проверяется на сервере.

## Вход

Откройте:

```text
http://<server>/login/
```

На панели входа нужно ввести логин и пароль. Доступные разделы определяются
правами пользователя и его групп.

Web-интерфейс не требует доступа в интернет после установки. Bootstrap хранится
локально в проекте и отдается через Django static / Nginx.

## Права

```text
accounts.view_reports      dashboard и reports
accounts.use_esp32_api     работа через ESP32 API
```

## Sidebar

В sidebar есть разделы:

- `Панель администратора` - ссылка на Django admin;
- `Отчеты` - текущие смены, произведенные детали, некорректные события.

`Панель администратора` видят пользователи с `is_staff=true`. Разделы внутри
Django admin видны по стандартным правам моделей `view/add/change/delete`.
`Отчеты` видят пользователи с `accounts.view_reports`.

## Главная страница

Главная страница показывает короткий dashboard:

- активные смены;
- проблемы связи;
- детали за сегодня;
- некорректные события за сегодня;
- количество станков, ESP32-устройств и пользователей;
- последние произведенные детали.

Пользователям без `accounts.view_reports` управленческая агрегация не
показывается.

## Справочники

Справочники ведутся через Django admin. В первой версии это самый короткий и
надежный путь: Django уже дает формы, поиск, фильтры, права и валидацию моделей.

Django admin оформлен через `django-jazzmin`. Это готовая тема поверх штатной
админки Django: проект не поддерживает отдельную самописную админку и не
переписывает стандартные формы вручную. Static-файлы темы входят в Python-пакет и
собираются командой `collectstatic`, поэтому после установки серверу не нужен CDN.

Основные справочники:

- пользователи, группы и права;
- станки;
- ESP32-устройства;
- разрешения работников на станки;
- расписания;
- типы деталей;
- состояния деталей.

## Dashboard и отчеты

Текущие смены:

```text
/dashboard/current-workers/
```

Отчет по деталям:

```text
/reports/details/
```

Отчет поддерживает фильтры и экспорт:

```text
/reports/details/export/csv/
/reports/details/export/xlsx/
```

Некорректные события:

```text
/reports/invalid-events/
```
