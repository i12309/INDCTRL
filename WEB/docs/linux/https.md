# HTTPS

В первой версии INDCTRL Nginx слушает HTTP на порту `80`. HTTPS не включен как обязательное требование, но его можно добавить поверх текущей схемы.

## Что потребуется

- Доменное имя или внутреннее DNS-имя сервера.
- TLS-сертификат.
- Приватный ключ сертификата.
- Настройка Nginx на порт `443`.
- Доверие сертификату на рабочих компьютерах и ESP32.

## Варианты сертификатов

### Самоподписанный сертификат

Подходит для тестового стенда или полностью изолированной сети. Минус: сертификат нужно вручную добавить в доверенные на компьютерах и ESP32.

### Внутренний центр сертификации предприятия

Хороший вариант для закрытой производственной сети. Администратор выпускает сертификат на внутреннее имя сервера, а корневой сертификат центра сертификации уже доверен рабочим компьютерам.

### Публичный сертификат

Подходит, если у сервера есть нормальное доменное имя и доступен выпуск сертификата через публичный центр сертификации, например Let's Encrypt.

## Где менять Nginx

Основной файл:

```text
docker/nginx/default.conf
```

В будущем можно добавить server block:

```nginx
server {
    listen 443 ssl;
    server_name indctrl.example.local;

    ssl_certificate /etc/nginx/certs/server.crt;
    ssl_certificate_key /etc/nginx/certs/server.key;

    # те же location, что и для HTTP
}
```

Сертификаты лучше передавать в контейнер через read-only volume, а не хранить в git.

## Важное для ESP32

Если ESP32 отправляет запросы по HTTPS, прошивка должна доверять сертификату сервера. Для самоподписанного или внутреннего сертификата это обычно требует добавить корневой сертификат в прошивку или хранилище доверия устройства.
