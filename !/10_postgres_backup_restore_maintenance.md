# Задача 10. Реализовать backup, restore и обслуживание PostgreSQL в Docker

## Цель

Сделать надежные скрипты и документацию для резервного копирования и восстановления PostgreSQL. Данные должны храниться долго, поэтому backup является обязательной частью production.

## Общие обязательные требования

- Проект называется `INDCTRL`.
- Язык: Python 3.12+.
- База данных: PostgreSQL.
- Запуск в production: Docker Compose на одном Linux-сервере.
- Сервисы:
  - `auth-service` — FastAPI-сервис авторизации работников на ESP32;
  - `event-service` — FastAPI-сервис приема событий о произведенных деталях;
  - `control-web` — Django-сервис, который содержит админку, dashboard и отчеты;
  - `postgres` — общая PostgreSQL-БД;
  - `nginx` — единая точка входа для HTTP.
- Админка и отчеты должны быть в одном Django-сервисе `control-web`. Внутри Django их надо разделить по приложениям/модулям.
- Django-модели и Django migrations являются основным источником структуры БД.
- FastAPI-сервисы работают с этой же БД напрямую через общий пакет и SQL-запросы, но не запускают миграции.
- Весь код должен быть понятен программисту, который плохо знает Python, FastAPI и Django.
- Все публичные классы, Django-модели, Pydantic-схемы, функции, методы и сервисные классы должны иметь докстринги или комментарии на русском языке.
- Сложная бизнес-логика должна быть пояснена комментариями на русском языке: проверки прав, расписаний, активных смен, идемпотентности, обработки дублей и ошибок.
- Комментарии должны объяснять смысл и причину решения, а не дублировать очевидное действие кода.
- В каждом сервисе должна быть собственная документация `README.md`: назначение, переменные окружения, запуск в Docker, основные endpoint'ы, диагностика и просмотр логов.

## Backup script

Создать:

```text
deploy/scripts/backup_postgres.sh
```

Скрипт должен:

```text
читать настройки из .env;
создавать директорию backups/postgres;
делать pg_dump из контейнера postgres;
создавать файл с датой и временем;
сжимать backup через gzip;
удалять старые backup-файлы старше заданного срока;
писать понятные сообщения в stdout;
возвращать ненулевой код при ошибке.
```

Пример имени файла:

```text
backups/postgres/machine_control_2026-05-17_030000.sql.gz
```

Пример команды внутри скрипта:

```bash
docker compose exec -T postgres pg_dump -U "$POSTGRES_USER" "$POSTGRES_DB" | gzip > "$BACKUP_FILE"
```

## Restore script

Создать:

```text
deploy/scripts/restore_postgres.sh
```

Скрипт должен:

```text
принимать путь к backup-файлу;
просить явное подтверждение перед восстановлением;
останавливать прикладные сервисы или предупреждать, что они должны быть остановлены;
восстанавливать dump в PostgreSQL;
писать понятные сообщения;
```

Восстановление должно быть документировано очень осторожно, потому что оно может перезаписать данные.

## Retention

В `.env.example` добавить:

```env
BACKUP_RETENTION_DAYS=30
BACKUP_DIR=./backups/postgres
```

## Автоматизация

Описать два варианта запуска backup:

```text
cron на хосте;
systemd timer на хосте.
```

Пример cron:

```cron
0 3 * * * cd /opt/INDCTRL && ./deploy/scripts/backup_postgres.sh >> /var/log/INDCTRL-backup.log 2>&1
```

## Проверка backup

В документации указать:

```text
backup считается рабочим только после тестового восстановления;
периодически надо проверять восстановление на отдельной машине или тестовой БД;
backup желательно копировать на другой диск или другой сервер.
```

## Maintenance

Создать `docs/backup/postgres-backup.md` и `docs/backup/postgres-restore.md`.

Дополнительно описать:

```text
как посмотреть размер БД;
как посмотреть размер таблицы details;
как посмотреть количество деталей по дням;
как проверить свободное место на диске;
```

Пример SQL:

```sql
SELECT pg_size_pretty(pg_database_size(current_database()));
SELECT count(*) FROM production_detail;
```

Названия таблиц уточнить по фактическим Django `db_table`.

## Критерии приемки

- Backup-скрипт создает `.sql.gz` файл.
- Restore-скрипт умеет восстановить backup в тестовую БД.
- В документации есть cron/systemd timer пример.
- Описан процесс проверки backup.
- Backup не хранит пароль в самом скрипте, а берет его из окружения или `.env`.
