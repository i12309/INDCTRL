# Задача для Codex: переделка web-интерфейса INDCTRL

Нужно переделать web-интерфейс INDCTRL в единый современный интерфейс на Django.

Интерфейс должен быть простым, приятным и понятным для пользователей предприятия.

## Цель

Сделать единый web-интерфейс, который адаптируется под роль пользователя.

Пользователь после входа должен попадать не в разрозненные страницы, а в один общий интерфейс с понятной навигацией.

## Авторизация

Должна быть панель авторизации.

На панели авторизации должен быть выбор роли.

После успешной авторизации пользователь попадает в единый интерфейс.

Интерфейс адаптируется под роль пользователя: недоступные функции не показываются.

Права доступа обязательно проверять на сервере. Нельзя ограничиваться только скрытием пунктов меню.

## Общий интерфейс

Сделать основной layout с sidebar-меню.

В sidebar должны быть разделы:

- Админка;
- Справочники;
- Отчеты.

Недоступные для роли разделы и функции нужно скрывать.

Интерфейс должен быть аккуратным, современным и удобным.

Если в проекте уже есть подходящие стили, использовать их.

Если готовых стилей нет, использовать последнюю стабильную версию Bootstrap.

## Главная страница

На главной странице нужен небольшой dashboard с агрегацией по данным.

Что именно показывать на dashboard, можно решить самостоятельно по текущим данным проекта.

Важно не усложнять. Нужен полезный, но простой dashboard без лишней аналитики и перегруза.

## Разделы

Разделы web-интерфейса должны быть связаны с текущей логикой проекта:

- пользователи и роли;
- станки;
- ESP32-устройства;
- разрешения и расписания;
- текущие смены;
- произведенные детали;
- некорректные события;
- отчеты.

Использовать стандартные возможности Django и Django admin там, где это сокращает код.

Не дублировать то, что уже удобно решается через Django admin.

## Стиль реализации

Код должен быть простым и понятным.

Не добавлять лишние frontend-фреймворки, сложные сборщики, SPA, отдельный frontend-проект или лишние абстракции.

Все тексты интерфейса, комментарии, docstring и документация должны быть на русском языке.

Комментарии нужны там, где есть бизнес-логика, проверка прав или неочевидное поведение.

## Документация

Обновить документацию по web-интерфейсу:

- как войти в систему;
- какие роли существуют;
- какие разделы доступны ролям;
- как пользоваться справочниками;
- как смотреть dashboard;
- как формировать отчеты.

Документация должна быть на русском языке.

## Критерий готовности

Пользователь входит через панель авторизации.

После входа открывается единый web-интерфейс.

Sidebar содержит разделы «Админка», «Справочники», «Отчеты».

Интерфейс адаптируется под роль пользователя.

Недоступные действия защищены на сервере.

Главная страница содержит простой dashboard.

Интерфейс выглядит современно и аккуратно.
