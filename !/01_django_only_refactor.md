# Задача для Codex: переход INDCTRL на Django-only

Нужно упростить проект INDCTRL и перевести его на Django-only архитектуру.

Текущие отдельные FastAPI-сервисы больше не нужны. Их функциональность нужно перенести в Django максимально просто и без лишних абстракций.

## Цель

Сделать проект проще, короче и легче для сопровождения.

Итоговая архитектура:

- Django-приложение;
- PostgreSQL;
- nginx;
- Docker Compose.

Django должен отвечать за:

- web-интерфейс;
- admin;
- dashboard;
- отчеты;
- API для ESP32;
- миграции БД.

## Основные требования

Сохранить существующую бизнес-логику:

- привязка ESP32-устройства к станку;
- авторизация работника на станке;
- создание рабочей смены;
- heartbeat;
- logout;
- прием события произведенной детали;
- защита от дублей;
- сохранение некорректных событий;
- роли и ограничения доступа;
- отчеты.

Удалить все, что стало лишним после перехода на Django-only:

- FastAPI-сервисы;
- отдельные Dockerfile для auth/event;
- лишние requirements;
- лишний общий код, если он больше не нужен;
- дублирующиеся схемы и ответы;
- лишние nginx upstream;
- документацию про старую микросервисную архитектуру.

## Стиль реализации

Код должен быть максимально простым и прямолинейным.

Не добавлять лишние слои, паттерны, брокеры, очереди, отдельные сервисы, use case/interactor/controller-архитектуру и другие усложнения.

Использовать стандартные возможности Django там, где это возможно.

Все комментарии, docstring, сообщения, названия и документация должны быть на русском языке.

Комментарии нужны в местах, где есть бизнес-логика или неочевидное решение. Очевидный код не нужно перегружать комментариями.

## Docker и запуск

Docker должен остаться.

Обновить Docker Compose под новую структуру:

- Django;
- PostgreSQL;
- nginx.

Проект должен запускаться просто и понятно.

## Документация

Полностью обновить документацию под Django-only архитектуру:

- README;
- запуск в Docker;
- переменные окружения;
- API для ESP32;
- структура проекта;
- описание резервного копирования;
- инструкция для установки на предприятии без интернета, если такая документация уже есть.

Документация должна быть на русском языке.

## Критерий готовности

Проект запускается в Docker.

FastAPI полностью удален из рабочей архитектуры.

API для ESP32 работает через Django.

Django управляет web-интерфейсом, admin, отчетами, API и миграциями.

Код стал проще, короче и понятнее.
