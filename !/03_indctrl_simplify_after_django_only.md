# Задача для Codex: точечное упрощение INDCTRL после перехода на Django-only

Нужно выполнить небольшую чистку проекта после перехода на Django-only.

Главная цель: сделать проект проще для установки, сопровождения и чтения кода. Не добавлять новые слои, новые фреймворки и лишние абстракции.

Все комментарии, тексты интерфейса, документация и сообщения должны быть на русском языке.

## 1. Убрать зависимость web-интерфейса от интернета

Сейчас Bootstrap подключается через CDN. Для предприятия без интернета это плохо.

Нужно сделать так, чтобы web-интерфейс полностью работал без доступа в интернет.

Bootstrap и нужные статические файлы должны лежать локально в проекте и отдаваться через Django static / nginx.

Документацию обновить: явно указать, что web-интерфейс не требует интернета после установки.

## 2. Упростить production Docker Compose под offline-установку

Production Compose не должен собирать образы на сервере предприятия.

Нужно разделить подход:

- локальная разработка может использовать `build`;
- production должен использовать заранее подготовленные Docker-образы;
- в production не должно быть попыток скачать или собрать приложение заново.

Для production использовать фиксированные имена и версии образов, без `latest`.

Документацию по offline-переносу образов обновить.

## 3. Удалить лишние переменные окружения

Проверить `.env.example`.

Если `DJANGO_PORT` реально нигде не используется, удалить его, чтобы не путать администратора.

Оставить только те переменные, которые реально влияют на работу проекта.

## 4. Makefile: минимум всего

Упростить Makefile.

Так как цель проекта — минимум лишнего, убрать команды, которые требуют дополнительных dev-зависимостей и сейчас не являются обязательными для запуска:

- `test`;
- `lint`;
- `format`.

Оставить только практические команды для эксплуатации:

- `build`;
- `up`;
- `down`;
- `logs`;
- `ps`;
- `migrate`;
- `createsuperuser`;
- `collectstatic`;
- `backup`.

Не добавлять отдельный dev requirements ради линтеров и тестов на этом этапе.

## 5. Упростить авторизацию

Убрать выбор роли из формы входа.

Пользователь должен вводить только логин и пароль.

Роль нужно брать из БД после успешной авторизации. Интерфейс должен адаптироваться под реальную роль пользователя:

- недоступные разделы не показывать;
- серверную проверку доступа обязательно оставить;
- скрытие пунктов меню не должно быть единственной защитой.

Документацию web-интерфейса обновить.

## 6. Комментарии и docstring

Комментарии на русском языке обязательны там, где есть бизнес-логика.

Но можно убрать очевидные комментарии и docstring, которые только повторяют название функции и не объясняют смысл.

Оставить комментарии там, где они реально помогают:

- почему нельзя доверять `userID`, `machineID`, `workID` из ESP32;
- почему один станок может иметь только одну активную смену;
- как работает защита от дублей деталей;
- зачем сохраняется `InvalidEvent`;
- как устроена проверка ролей;
- почему API отключает CSRF для ESP32.

## Критерий готовности

Проект стал проще и понятнее.

Web-интерфейс работает без интернета.

Production Compose рассчитан на готовые offline-образы.

Форма входа стала проще: только логин и пароль.

Makefile содержит только эксплуатационные команды.

Документация обновлена под фактическое состояние проекта.
