# Web-интерфейс INDCTRL

INDCTRL использует единый Django-интерфейс. После входа пользователь попадает в
общую оболочку с sidebar-меню. Разделы скрываются по роли, а доступ дополнительно
проверяется на сервере.

## Вход

Откройте:

```text
http://<server>/login/
```

На панели входа нужно ввести логин и пароль. Роль берется из БД после успешной
авторизации.

Web-интерфейс не требует доступа в интернет после установки. Bootstrap хранится
локально в проекте и отдается через Django static / Nginx.

## Роли

```text
admin      admin, справочники, dashboard, reports
director   dashboard и reports
manager    dashboard и reports
worker     вход в систему без управленческих разделов
```

## Sidebar

В sidebar есть разделы:

- `Админка` - ссылка на Django admin;
- `Справочники` - пользователи, роли, станки, ESP32, разрешения, расписания,
  типы и состояния деталей;
- `Отчеты` - текущие смены, произведенные детали, некорректные события.

`Админка` и `Справочники` видит только роль `admin`. `Отчеты` видят `admin`,
`director`, `manager`.

## Главная страница

Главная страница показывает короткий dashboard:

- активные смены;
- проблемы связи;
- детали за сегодня;
- некорректные события за сегодня;
- количество станков, ESP32-устройств и пользователей;
- последние произведенные детали.

Для роли `worker` управленческая агрегация не показывается.

## Справочники

Справочники ведутся через Django admin. В первой версии это самый короткий и
надежный путь: Django уже дает формы, поиск, фильтры, права и валидацию моделей.

Основные справочники:

- пользователи и роли;
- станки;
- ESP32-устройства;
- разрешения работников на станки;
- расписания;
- типы деталей;
- состояния деталей.

## Dashboard и отчеты

Текущие смены:

```text
/dashboard/current-workers/
```

Отчет по деталям:

```text
/reports/details/
```

Отчет поддерживает фильтры и экспорт:

```text
/reports/details/export/csv/
/reports/details/export/xlsx/
```

Некорректные события:

```text
/reports/invalid-events/
```
